-- What Makes GLM-5 Unique --

DSA: DeepSeek Sparse Attention

How the indexer works

1. Query: q_resid [B,S,2048] -> wq_b -> [B,S,32*128] -> partial RoPE -> concat
2. Key: hidden [B,S,6144] -> wk -> [B,S,128] -> LayerNorm -> partial RoPE -> concat
3. Score: einsum(q,k) * scale -> ReLU -> weighted sum across 32 heads -> [B,S,T]
4. Select: torch.topk(scores, k=2048) -> sparse indices [B,S,2048]
5. Mask: Scatter indices into -inf mask, combine with causal mask

Result: O(n * 2048) instead of O(n^2) -- 1.5-2x speedup at 128K+ context

Why DSA beats alternatives

vs Sliding Window: SWA loses 30 pts on RULER@128K. DSA is lossless.
vs Linear Attention: GDN/SimpleGDN degrade on retrieval. DSA doesn't.
Key insight: 90% of attention entries at 128K are redundant

Critical RL fix: Non-deterministic CUDA topk broke RL in a few steps.
Switching to deterministic torch.topk solved it. (Paper Section 3.2)

RL strategy: Indexer params are frozen during RL -- only the main model trains.

MLA: Multi-Latent Attention

KV compression path

Standard GQA cache: 128 dims * 8 KV heads = 1024 per token
MLA cache: 512 latent + 64 rope = 576 per token (44% smaller)

Compress: hidden(6144) -> kv_a_proj -> latent(512) + rope(64)
Expand: latent(512) -> kv_b_proj -> K_nope(192*64) + V(256*64)

Q path: hidden(6144) -> q_a(2048) -> RMSNorm -> q_b(16384=64*256)

GLM-5 MLA innovations

Muon Split: Orthogonalize per-head, not per-matrix
Standard MLA + Muon couldn't match GQA-8. Per-head orthogonalization fixes it.

Head dim: Increased 192 -> 256, reduced heads by 1/3
Same training FLOPs but lower decode compute.

Stable logits: With Muon Split, attention logit scale stays stable without clipping.

256-Expert MoE with Sigmoid Routing

Routing mechanism

1. Linear projection -> per-expert scores
2. Sigmoid activation (not softmax)
3. Add e_score_correction_bias (load balancing, no aux loss)
4. Group scores -> select top groups -> top-8 within groups
5. Normalize weights, scale by 2.5x

Why sigmoid? Independent expert scoring (softmax forces competition)

Expert storage

Format: Stacked 3D tensors
gate_up_proj: [256, 4096, 6144]
down_proj: [256, 6144, 2048]

Reference: Per-expert Python loop (model.py)
Accelerated: Triton grouped_gemm in unsloth_moe/
Covers ~60% of total FLOPs across 75 MoE layers
6 features that distinguish GLM-5 from Llama, Qwen, Mistral, and other architectures:

1. DSA -- Learned sparse token selection. No other family uses this. Makes 200K context affordable.
2. MLA -- KV compressed to 576-dim latent instead of full per-head caching. 44% less memory than GQA-8.
3. 256 fine-grained experts -- Most MoE models use 8-64. Finer specialization at same active budget.
4. Sigmoid routing -- Independent expert scoring, not softmax competition. With group selection + 2.5x scale.
5. MTP param sharing -- 3 draft layers share 1 weight set. Acceptance rate 2.76 (vs DeepSeek's 2.55).
6. Deterministic top-k for RL -- CUDA non-deterministic topk broke training. torch.topk fixed it. Unique finding.
